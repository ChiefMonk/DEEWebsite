Classical Reasoning

Knowledge Representation and Reasoning expresses reasoning in terms of formal languages called logics. While there are many logics, we focus on propositional logic and use classical propositional logic as the basis for extension to KLM-style defeasible reasoning. In classical propositional logic, we can encode objects such as penguins and birds as propositional atoms and use binary connectives to build more complex statements called formulas. A notion of entailment then enables us to infer new knowledge from existing knowledge: A set of statements—such as the LHS of the statement above—is referred to as a knowledge base since it represents knowledge we have modeled. Modeling information in a knowledge base and using classical entailment to infer information, in the manner demostrated above, provides us with a simple reasoning system.

Explanation

Sometimes we want to know why certain entailments hold. For classical logic, explanation tells us which statements in our knowledge base are responsible for a given entailment. In our work, we focus on a form of explanation known as justification. Justifications are minimal subsets of statements that allow us to infer a conclusion. For example, suppose we know the unrelated information that ‘birds have eyes’: The entailment that ‘penguins fly’ for the knowledge base above then has the justification since this amounts to 'just enough' information to allow for the inference that ‘penguins fly’. This illustrates how justifications correspond with how people explain inferences when reasoning: we certainly would not state ‘birds have eyes’ when substantiating why ‘penguins fly’.

Explanation has been shown to be an important aspect of reasoning systems. The concept is particularly useful for knowledge base debugging but has also been shown to be important for user comprehension and confidence in reasoning systems. Thus they are an key part of using reasoning systems practically, making them an important concept to extend to defeasible reasoning.
